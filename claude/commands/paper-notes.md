# Paper Notes Template

Create structured notes for the paper the user is reading. Ask for the paper title/PDF/link if not provided.

Generate a markdown file with the following structure:

```markdown
# [Paper Title]

**Authors**: [Authors]
**Venue**: [Conference/Journal] [Year]
**Paper link**: [URL if available]
**Code**: [GitHub link if available]

## Summary (3-5 sentences)
[High-level summary of what the paper does and why it matters]

## Problem & Motivation
- What problem does this solve?
- Why is it important?
- What are limitations of prior work?

## Method
- Core technical approach
- Key innovations
- Architecture/algorithm details

## Experiments
- Datasets used
- Baselines compared
- Main results
- Ablation studies

## Key Findings
- What are the main takeaways?
- What works? What doesn't?
- Any surprising results?

## Strengths
- What does this paper do well?

## Limitations
- What's missing or unclear?
- What assumptions are made?
- Where might this fail?

## Related Work
- Key papers cited
- How this fits in the literature

## Relevance to My Work
- Why am I reading this?
- How might I use these ideas?
- What should I try?

## Questions & Follow-ups
- What remains unclear?
- What should be investigated further?

## Implementation Notes
[If planning to implement - what's needed]
```

Fill in sections based on the paper content. Be specific with page numbers for key claims.
